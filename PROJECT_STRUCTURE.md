## Описание каждого файла

### SKILL.md ⭐ (ГЛАВНЫЙ)

**Размер:** ~330 строк  
**Назначение:** Основной файл скилла для OpenCode  
**Формат:** YAML frontmatter + Markdown тело

**Содержит:**
- Frontmatter с метаданными (name, description)
- Роль агента (что он делает)
- Ключевые принципы работы
- Пошаговые инструкции
- Полный справочник API Яндекс Трекера
- Примеры JSON тел запросов
- Обработку ошибок
- Дисциплину вывода

**Когда используется:**
- Загружается в контекст агента при запросе `skill()`
- Полностью становится инструкциями агента
- Агент следует этим инструкциям при работе

### README.md 📖

**Размер:** ~200 строк  
**Назначение:** Главная страница проекта

**Содержит:**
- Описание того, что делает скилл
- Примеры использования
- Инструкции быстрого старта
- Информацию о безопасности
- Требования
- Ссылки на другие файлы

**Кто читает:** Первый раз люди на GitHub

### INSTALL.md 🚀

**Размер:** ~300 строк  
**Назначение:** Подробные инструкции установки

**Содержит:**
- 3 способа установки (локально, через git, npm)
- Требования к окружению
- Как получить токен Трекера
- Отладка проблем установки
- Как обновить скилл
- Как удалить скилл

**Кто читает:** Люди, которые хотят установить скилл

### EXAMPLES.md 💡

**Размер:** ~500 строк  
**Назначение:** Реальные примеры использования

**Содержит:**
- 5 примеров разных процессов (от простого к сложному)
- Для каждого примера: описание, уточнения, результат
- Типичные вопросы от агента
- Советы и трюки
- Примеры результата

**Кто читает:** Люди, которые хотят понять, как использовать

### CONTRIBUTING.md 👥

**Размер:** ~400 строк  
**Назначение:** Инструкции для контрибьютеров

**Содержит:**

- Стандарты качества
- Процесс PR
- Правила формата
- Версионирование

**Кто читает:** Люди, которые хотят улучшить проект

### LICENSE 📄

**Назначение:** Юридическая лицензия

**Содержит:**

## Как работает доставка в GitHub

```
1. Клонируешь репо
   ↓
2. Читаешь README.md
   ↓
3. Следуешь INSTALL.md
   ↓
4. Смотришь EXAMPLES.md
   ↓
5. Используешь скилл в OpenCode
   ↓
6. Если хочешь улучшить → CONTRIBUTING.md
   ↓
7. Создаешь PR
   ↓
8. GitHub Actions запускает validate-skill.yml
   ↓
9. После одобрения → merge
```

## Почему так структурировано

### 🎯 Цель: минимальная сложность

- Только необходимые файлы
- Каждый файл - четкая роль
- Не дублируем информацию

### 🔒 Цель: безопасность

- SKILL.md с инструкцией "никогда не раскрывай кредентиалы"
- .gitignore защищает от утечек
- GitHub Actions проверяет каждый PR

### 📖 Цель: документирование

- README для "что это"
- INSTALL для "как начать"
- EXAMPLES для "как использовать"
- CONTRIBUTING для "как улучшать"

### ✅ Цель: качество

- GitHub Actions - автоматическая проверка
- CONTRIBUTING - стандарты
- Простота для контрибьютинга
